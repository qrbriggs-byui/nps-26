Accessible Design Example

This page gives a simple example of the POUR principles from WCAG 2.2 and shows how ARIA state can be updated on a basic dialog box.

POUR Principles

Perceivable

Content should be available to more than one sense. This example uses real text, strong color contrast, and descriptive alternative text.

This is a cute cat looking at the camera.

Operable

Users should be able to interact with the page using a keyboard. Try the skip link, tab through the controls, and use Escape to close the dialog.

Jump to the dialog example

Understandable

Content and controls should be clear. This form field has a visible label and plain instructions.

Robust

Code should work well with assistive technology. This page uses landmarks, headings, labels, and ARIA attributes that update when the dialog opens and closes.
